Fase 00: O Cenário

🛡️ Sua Fortaleza Digital

Um site não é apenas código; é um território. Sem os Headers de Segurança, seu site é um castelo de portas abertas no meio de uma floresta de perigos.

👤

O Infiltrado (XSS)

O vilão que tenta "ouvir" o que o usuário digita (senhas, cartões) injetando um script espião.

Muro: CSP
🖼️

O Sequestrador

Aquele que coloca o seu site dentro de um quadro invisível para roubar cliques (Clickjacking).

Muro: X-Frame
🕵️

O Xereta (MIME)

O vilão que tenta convencer o navegador a ler um arquivo de imagem como se fosse um vírus.

Muro: nosniff
Fase 01: Os Porteiros

💂‍♂️ Quem protege seu castelo?

Headers são instruções enviadas pelo servidor que dizem ao navegador como ele deve se comportar ANTES de abrir o site.

O General: CSP

O Content Security Policy decide quais scripts, imagens e estilos podem carregar.

  • 'self': "Só confio em mim".
  • -src: Significa Fonte (Source).
script-src Controla os arquivos JS. Evita que hackers injetem scripts que roubam senhas.
style-src Controla o visual (CSS). Impede que alguém mude o layout para te enganar.
img-src Controla as imagens. Evita fotos de servidores maliciosos e pixels espiões.

🕵️‍♂️ Anatomia de um Ataque

Como alguém infecta seu site? Pela "carona" em campos de comentários ou buscas.

1. A Brecha

O hacker escreve <img src="virus.jpg"> no seu chat.

2. A Armadilha

O navegador do seu cliente baixa a imagem do hacker automaticamente.

3. O Bloqueio

Com CSP, o navegador diz: "Essa fonte não é autorizada! BLOQUEADO!".

Fase 02: Construção

🏗️ Erguendo os Muros (Cloudflare)

Use as Transform Rules para injetar esses headers no seu domínio.

Filtro: Hostname contains seusite.com
Ação: Set Static Header
Content-Security-Policy = default-src 'self';
X-Content-Type-Options = nosniff

⚠️ Alerta: Unsafe-Inline

Nunca use 'unsafe-inline'. Mova seus scripts para arquivos .js separados. Isso é o que garante o A+.

❌ Inseguro onclick="acao()"
✅ Seguro addEventListener

✅ Dica: Use sempre SET

O comando Set Static garante que sua regra nova apague qualquer configuração velha ou errada.

Fase 03: Inspeção

🥷 Verificação Ninja

Não confie no Mozilla enquanto estiver configurando. Use o terminal ou o DevTools.

O Terminal (Curl)

curl -I https://seusite.com?v=1
O ?v=1 quebra o cache e mostra a verdade.

O Navegador (F12)

Aba Network > Response Headers. Se o header estiver lá, você venceu!

Dica do Histórico: Mesmo que o site diga "Just Now", ele pode estar te mostrando cache de meses atrás. Confira sempre o Last Scan no histórico!
Fase 04: O Olimpo

🏛️ Além do A+

O que separa os iniciantes dos verdadeiros mestres da segurança web?

Dilema dos Gigantes

Sites como Supabase ou Vercel às vezes tiram notas menores (60/70) porque são muito complexos. Você, por ser ágil, conseguiu a perfeição técnica.

Nível Google (Trusted Types)

O Google usa Trusted Types. É uma trava que impede que qualquer texto comum vire script sem um atestado de higiene total.
