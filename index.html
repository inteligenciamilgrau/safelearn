00

Por que estamos aqui? 🕵️‍♂️

Antes de apertar botões, entenda quem são os "vilões" que o Mozilla Observatory quer que você vença:

👺

O Infiltrado (XSS)

Ele tenta colocar um código "espião" dentro do seu site para roubar senhas dos seus usuários.

Defesa: CSP
🕸️

A Fachada Falsa

O hacker "embrulha" seu site dentro de outro para enganar as pessoas e fazê-las clicar onde não deviam.

Defesa: X-Frame-Options
👂

O Fofoqueiro

Ele fica "ouvindo" a conversa entre o usuário e o seu site. Se não estiver blindado, ele rouba tudo.

Defesa: HSTS
Moral da história: O Mozilla Observatory é o seu Auditório de Segurança. Ele testa se você instalou as trancas certas para expulsar esses vilões.

Conheça seus Defensores

Clique nos cards abaixo para entender o que cada header faz.

🛡️

CSP

O mestre do controle. Define o que pode rodar no seu site.

Previne ataques de Cross-Site Scripting (XSS).
🔒

HSTS

Força a conexão segura via HTTPS sempre.

Impede ataques de interceptação (MITM).
🖼️

X-Frame

Bloqueia seu site em iframes maliciosos.

Protege contra Clickjacking.

03

O Código Mestre

Copie esta configuração e aplique no seu servidor (Cloudflare, Nginx, Apache).

SECURITY HEADERS CONFIG 
Content-Security-Policy: default-src 'self';
Strict-Transport-Security: max-age=63072000; preload
X-Frame-Options: DENY
X-Content-Type-Options: nosniff
Referrer-Policy: strict-origin-when-cross-origin
Permissions-Policy: camera=(), microphone=()
04

Onde eu colo isso? 📍

Estes headers não vão no HTML. Eles vão na configuração do seu Host.

Cloudflare

Vá em Rules > Transform Rules e crie uma regra de "Modify Response Header".

Vercel

Crie um arquivo vercel.json na raiz do projeto com a chave "headers".

Apache (.htaccess)

Adicione: Header set X-Frame-Options "DENY" no seu arquivo .htaccess.

05

Tradução para Humanos (CSP) 🧑‍🏫

O CSP é o "Porteiro" do seu site. Veja como dar ordens para ele de um jeito simples:

O que vigiar? O que significa? Quem pode entrar? (Exemplos)
A Regra Geral
default-src		 Vale para tudo que você não explicou abaixo. 'self'
(Apenas meu próprio site)
Scripts (Ajudantes)
script-src		 Arquivos que fazem o site "funcionar" (JS). https://google.com
(Só o Google tá liberado)
Estilos (Roupas)
style-src		 Cores e fontes que deixam o site bonito (CSS). 'unsafe-inline'
(Código escrito direto no HTML)
Imagens
img-src		 Fotos, logos e ícones. 'self' data:
(Minhas fotos e ícones simples)
Porta-Retrato
frame-src		 Outros sites aparecendo dentro do seu. https://youtube.com
(Apenas vídeos do YouTube)
💡 Dica do Porteiro: Sempre que ler 'self', pense em "Eu Mesmo". É o seu site dizendo: "Eu confio em mim, mas não confio em estranhos!".

Por que isso não é automático? 🤔

Se essas travas são tão boas, por que os navegadores já não deixam elas ligadas para todo mundo?

A Regra do "Não Quebrar"

Se o Chrome ligasse o CSP máximo hoje, metade dos sites do mundo (incluindo sites de governo e bancos antigos) parariam de funcionar. O navegador precisa aceitar o "jeito antigo" para a internet não colapsar.

Segurança é uma Escolha

Ao configurar esses headers, você está dando um sinal para o navegador: "Ei, eu limpei meu código e meu site é moderno. Pode aplicar a segurança máxima aqui!"

Facilidade vs. Rigor

Muitos sites são feitos "nas pressas". Manter tudo organizado e seguro exige mais tempo e estudo. O Mozilla Observatory premia quem decide fazer o caminho mais difícil, porém correto.

👑 Header vs 📜 Meta Tag

HTTP Header

O segurança barra na porta da frente. Protege o site ANTES dele carregar. É o que o Mozilla exige para o A+.

Meta Tag

É um aviso colado na parede lá dentro. O script malicioso pode rodar antes do navegador ler a tag. Não suporta HSTS.
