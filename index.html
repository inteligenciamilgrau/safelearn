03

O Código Mestre

Copie esta configuração e aplique no seu servidor (Cloudflare, Nginx, Apache).

SECURITY HEADERS CONFIG 
Content-Security-Policy: default-src 'self';
Strict-Transport-Security: max-age=63072000; preload
X-Frame-Options: DENY
X-Content-Type-Options: nosniff
Referrer-Policy: strict-origin-when-cross-origin
Permissions-Policy: camera=(), microphone=()
04

Onde eu colo isso? 📍

Estes headers não vão no HTML. Eles vão na configuração do seu Host.

Cloudflare

Vá em Rules > Transform Rules e crie uma regra de "Modify Response Header".

Vercel

Crie um arquivo vercel.json na raiz do projeto com a chave "headers".

Apache (.htaccess)

Adicione: Header set X-Frame-Options "DENY" no seu arquivo .htaccess.

05

O que cada termo significa? (Dicionário Real) 📖

O CSP usa termos técnicos que parecem grego. Vamos traduzir para a língua da gente:

O segredo do "-src"

Quase todos os termos terminam em -src. Isso é abreviação de Source (Fonte/Origem). Ele define de ONDE o navegador pode baixar as coisas.

script-src Controla a fonte dos arquivos JavaScript (os "ajudantes" que fazem o site funcionar). Evita que hackers injetem scripts que roubam senhas.
style-src Controla a fonte dos CSS (as "roupas" do site). Evita que alguém mude o visual do seu site para te enganar.
img-src Controla de onde vêm as Imagens. Evita o carregamento de fotos de servidores maliciosos.
default-src É a Regra Geral. Se você não deu uma ordem específica para um tipo de arquivo, o navegador usa essa regra aqui como segurança.
frame-ancestors Controla quem pode ser o "pai" do seu site em um quadro. Evita que o seu site seja "sequestrado" dentro de outro site falso.
💡 Por que usar isso? Para evitar o Roubo de Dados. Quando você diz script-src 'self', o navegador só aceita scripts que moram no seu próprio site. Se um hacker tentar injetar um script de fora, o navegador bloqueia na hora!

🔗 Como combinar tudo? Use o ponto e vírgula ; para separar as ordens. Exemplo: default-src 'self'; frame-ancestors 'none';

🕵️‍♂️ Anatomia de um Ataque (Na Prática)

Você deve estar pensando: "Como alguém colocaria algo no MEU site se o servidor é meu?"

1. A Brecha:

O hacker vai até o seu campo de comentários ou busca e digita:
<img src="http://hacker.com/pixel-espiao">

2. A Armadilha:

Seu site salva esse comentário. Agora, todo visitante que abrir a página vai, sem saber, baixar a imagem do hacker.

3. O Roubo:

Através dessa "simples imagem", o hacker descobre o IP do visitante, onde ele mora e pode até tentar infectar o navegador dele.

🛡️ O que o CSP faz?

O navegador lê o comentário do hacker, olha para o seu CSP e diz: "Opa! Essa imagem vem de um site estranho e a dona do site só me autorizou carregar coisas do próprio site ('self'). BLOQUEADO!"

⚠️ O Perigo do 'unsafe-inline'

Muitos tutoriais mandam você adicionar 'unsafe-inline' para "consertar" erros no console. NÃO FAÇA ISSO!

❌ Jeito Inseguro 
<button onclick="alerta()">Clique</button>
O hacker pode injetar comandos aqui.
✅ Jeito Seguro (A+) 
btn.addEventListener('click', alerta);
Código separado em arquivo .js

Para ganhar o A+, seu CSP deve conter apenas 'self'. Se precisar de scripts externos, coloque a URL do site (ex: https://google.com).

Por que isso não é automático? 🤔

Se essas travas são tão boas, por que os navegadores já não deixam elas ligadas para todo mundo?

A Regra do "Não Quebrar"

Se o Chrome ligasse o CSP máximo hoje, metade dos sites do mundo (incluindo sites de governo e bancos antigos) parariam de funcionar. O navegador precisa aceitar o "jeito antigo" para a internet não colapsar.

Segurança é uma Escolha

Ao configurar esses headers, você está dando um sinal para o navegador: "Ei, eu limpei meu código e meu site é moderno. Pode aplicar a segurança máxima aqui!"

Facilidade vs. Rigor

Muitos sites são feitos "nas pressas". Manter tudo organizado e seguro exige mais tempo e estudo. O Mozilla Observatory premia quem decide fazer o caminho mais difícil, porém correto.

👑 Header vs 📜 Meta Tag

HTTP Header

O segurança barra na porta da frente. Protege o site ANTES dele carregar. É o que o Mozilla exige para o A+.

Meta Tag

É um aviso colado na parede lá dentro. O script malicioso pode rodar antes do navegador ler a tag. Não suporta HSTS.

07

Dica Ninja: Teste via Terminal 🥷

Não quer esperar o Mozilla? Teste seus headers agora mesmo usando o terminal do seu PC.

POWERSHELL / TERMINAL 
curl -I https://seusite.com

O comando -I mostra apenas o "envelope" do site. Se você vir suas regras na lista que aparecer, significa que está funcionando!

08

Meu teste não atualiza! 🔄

O Mozilla Observatory guarda o resultado por 24h. Se você mudou algo e a nota continua igual, siga estes passos:

1. Force Rescan

Ao digitar seu domínio, clique no botão "Force Rescan". Isso obriga o Mozilla a olhar seu site de novo agora mesmo.

2. Verifique a Data

Olhe no topo do teste se a data do "Last Scan" é a de hoje. Se for antiga, você está vendo um "fantasma" do passado.

3. Use o Terminal

Se o curl -I mostrar o header, o erro é do Mozilla. Confie no seu terminal, ele não mente!
